Squid Monitor

Introduction

The Proxy Monitor module provides a dashboard for monitoring your Squid proxy server. It focuses on blocked requests (TCP_DENIED) and provides detailed statistics about ACL activity.

Features

  • Real-time statistics from Squid access logs
  • Monitor TCP_DENIED (blocked) requests
  • Track multiple ACL files and their block counts
  • Support for file-based and regex-based ACLs
  • Detailed blocked requests by client IP with expandable details
  • Top blocked domains and clients
  • Configurable time ranges and auto-refresh
  • Zero external dependencies (pure Perl parsing)

Configuration

In the module configuration, you can customize:

  • Log file path: Location of Squid's access.log (default: /var/log/squid/access.log)
  • Lines to parse: How many log lines to read (default: 50,000)
  • Time range: Only show requests from last X hours (default: 24)
  • ACL files: List of ACL files to monitor with custom labels
  • Auto-refresh: Automatically reload the dashboard every X seconds

ACL Monitoring

You can monitor two types of ACLs:

File-based ACLs

Format:

/path/to/acl.txt=Label Name

Example:

/etc/acl/acl_squid/social_media.txt=Social Media
/etc/acl/acl_squid/streaming.txt=Streaming Sites
/etc/acl/acl_squid/adult_content.txt=Adult Content

Regex-based ACLs

Format:

regex:pattern=Label Name

Example:

regex:^(http|https)://[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+=Block IPv4
regex:.*\.(exe|msi|dmg)$=Executable Files

The dashboard will show how many blocks each ACL generated and identify which ACL blocked each request.

Blocked Requests Detail View

The "Blocked Requests by Client IP" section shows all clients with blocked requests. Click on any client IP to expand and see:

  • Date and time of each blocked request
  • Domain or IP that was blocked
  • Which ACL performed the block

Performance Notes

  • Parsing is done in real-time when you load the page
  • Large log files may take a few seconds to parse
  • Adjust "Lines to parse" if the page loads slowly
  • Squid automatically rotates logs, so access.log stays manageable
  • Regex ACLs are slightly slower than file-based ACLs
