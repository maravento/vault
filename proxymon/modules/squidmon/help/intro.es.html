Monitor de Squid

Introducción

El módulo Monitor de Proxy proporciona un dashboard para monitorear su servidor proxy Squid. Se enfoca en peticiones bloqueadas (TCP_DENIED) y proporciona estadísticas detalladas sobre la actividad de las ACL.

Características

  • Estadísticas en tiempo real desde los logs de Squid
  • Monitoreo de peticiones TCP_DENIED (bloqueadas)
  • Seguimiento de múltiples archivos ACL y sus conteos de bloqueos
  • Soporte para ACLs basadas en archivos y regex
  • Peticiones bloqueadas detalladas por IP de cliente con detalles expandibles
  • Dominios y clientes más bloqueados
  • Rangos de tiempo configurables y auto-actualización
  • Cero dependencias externas (parsing puro en Perl)

Configuración

En la configuración del módulo, puede personalizar:

  • Ruta del archivo de log: Ubicación del access.log de Squid (predeterminado: /var/log/squid/access.log)
  • Líneas a procesar: Cuántas líneas del log leer (predeterminado: 50,000)
  • Rango de tiempo: Solo mostrar peticiones de las últimas X horas (predeterminado: 24)
  • Archivos ACL: Lista de archivos ACL a monitorear con etiquetas personalizadas
  • Auto-actualización: Recargar automáticamente el dashboard cada X segundos

Monitoreo de ACL

Puede monitorear dos tipos de ACLs:

ACLs basadas en archivos

Formato:

/ruta/a/acl.txt=Nombre Etiqueta

Ejemplo:

/etc/acl/acl_squid/redes_sociales.txt=Redes Sociales
/etc/acl/acl_squid/streaming.txt=Sitios de Streaming
/etc/acl/acl_squid/contenido_adulto.txt=Contenido Adulto

ACLs basadas en regex

Formato:

regex:patrón=Nombre Etiqueta

Ejemplo:

regex:^(http|https)://[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+=Bloquear IPv4
regex:.*\.(exe|msi|dmg)$=Archivos Ejecutables

El dashboard mostrará cuántos bloqueos generó cada ACL e identificará qué ACL bloqueó cada petición.

Vista Detallada de Peticiones Bloqueadas

La sección "Peticiones Bloqueadas por IP de Cliente" muestra todos los clientes con peticiones bloqueadas. Haga clic en cualquier IP de cliente para expandir y ver:

  • Fecha y hora de cada petición bloqueada
  • Dominio o IP que fue bloqueado
  • Qué ACL realizó el bloqueo

Notas de Rendimiento

  • El análisis se hace en tiempo real cuando carga la página
  • Archivos de log grandes pueden tardar unos segundos en procesarse
  • Ajuste "Líneas a procesar" si la página carga lentamente
  • Squid rota los logs automáticamente, así que access.log se mantiene manejable
  • Las ACLs regex son ligeramente más lentas que las ACLs basadas en archivos
